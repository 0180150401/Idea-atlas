# Pitfalls Research: Idea Atlas

## Pitfall: Building a Quote Database Instead of an Atlas

- **Warning signs**: Search and CRUD work, but relationships, worldview axes, and generation context are missing.
- **Prevention**: Make thought-object metadata, edges, and provenance part of the earliest schema.
- **Phase to address**: Phase 1 and Phase 2.

## Pitfall: Treating Aphorisms as Short Summaries

- **Warning signs**: Outputs are polished paraphrases without compression, form, reversal, or interpretive surplus.
- **Prevention**: Track rhetorical forms and require bundle outputs with counter-aphorism, gloss, reversal, hostile reading, and provenance.
- **Phase to address**: Phase 3.

## Pitfall: Fake Profundity

- **Warning signs**: Generated lines sound wise but cannot explain lineage, tension, or usefulness.
- **Prevention**: Require provenance, hostile reading, and rubric evaluation for each generated bundle.
- **Phase to address**: Phase 3 and Phase 4.

## Pitfall: Copyright and Quote Leakage

- **Warning signs**: Generated output repeats distinctive source wording or structure too closely.
- **Prevention**: Store provenance, separate original text from metadata, and run similarity checks before public use.
- **Phase to address**: Phase 4.

## Pitfall: Overbuilding Graph Infrastructure

- **Warning signs**: Time goes into graph tooling before the user can ingest, retrieve, generate, and evaluate.
- **Prevention**: Use relational edge tables first; revisit dedicated graph storage only after relationship workflows are proven.
- **Phase to address**: Phase 1.

## Pitfall: Map-First Product Risk

- **Warning signs**: Visual clustering becomes the main build effort before the intellectual workflow works.
- **Prevention**: Build text-first region selection, search, and generation before the full 2D atlas.
- **Phase to address**: Phase 2 and Phase 5.

## Pitfall: Worldview Axes Too Generic

- **Warning signs**: Metadata looks like generic tags and does not reflect the user's recurring tensions.
- **Prevention**: Start with a small editable axis set and make axis calibration part of review.
- **Phase to address**: Phase 2.
