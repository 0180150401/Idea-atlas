# Feature Research: Idea Atlas

## Table Stakes

- **Ingestion**: User can add raw notes, quotes, annotations, clipped passages, and original reflections.
- **Atomic Thought Objects**: User can break material into propositions, distinctions, metaphors, warnings, questions, observations, counterexamples, and aphorisms.
- **Structured Metadata**: User can store source, provenance, topical tags, claim type, stance, target, rhetoric, imagery, emotional valence, metaphor family, and worldview coordinates.
- **Semantic Retrieval**: User can search by meaning, topic, worldview axis, and metadata filters.
- **Relationships**: User can express supports, rebuts, extends, inverts, and descends-from edges between items.
- **Atlas Navigation**: User can browse clusters, neighborhoods, border zones, and outliers.
- **Bundle Generation**: User can generate an aphorism bundle from a selected region of the atlas.
- **Provenance Display**: User can see which notes influenced an output.
- **Evaluation Loop**: User can score generated outputs for novelty, fidelity, interpretive depth, quote leakage, and usefulness.

## Differentiators

- **Worldview Coordinates**: Personal philosophical axes guide retrieval and generation.
- **Rhetorical Form Modeling**: Aphoristic form is treated as data, not style polish.
- **Tension-Preserving Output**: Counter-aphorisms, reversals, and hostile readings prevent one-dimensional output.
- **Visual Liminal Map**: The atlas makes semantic neighborhoods and edge zones visible.
- **Conceptual Blending Workflow**: Near-neighbor retrieval plus distant adjacent material supports original recombination.

## Anti-Features

- **Social feed**: Distracts from personal intellectual workflow.
- **Generic AI writing assistant**: The product should be atlas-grounded, not blank-page prompting.
- **Unattributed quote remixing**: Creates legal, ethical, and quality risk.
- **Overbuilt graph platform**: Too much infrastructure before the thought-object workflow is proven.

## Complexity Notes

The highest-risk features are structured extraction consistency, worldview-axis design, output originality checks, and useful visual projection. Basic ingestion and metadata browsing should be kept simple so the first end-to-end loop ships quickly.
