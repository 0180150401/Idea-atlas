# v1.1 Concept Synthesis: Personal Literary Intelligence

## Product Reframe

Idea Atlas should evolve from an idea/aphorism atlas into a personal literary intelligence system: part commonplace book, part close-reading engine, part influence graph, part constrained generative workshop.

The goal is not to generate generic literary prose. The goal is to help the user metabolize influence into original voice.

## Three Primary Sections

### 1. Living Canon

The user builds a private literary memory from books, passages, marginalia, notes, drafts, media, and personal reactions.

This section must minimize manual input. The system should infer bibliographic, textual, rhetorical, affective, stylistic, and provenance metadata wherever possible.

### 2. Influence Atlas

The canon becomes a live node network of authors, works, passages, motifs, rhetorical forms, worldview tensions, affective registers, relationships, contradictions, and creative routes.

The graph should not merely show semantic similarity. It should show literary influence: echoes, rebuts, extends, inverts, parodies, descends-from, radicalizes, secularizes, shares-image-system, and shares-rhetorical-form.

### 3. Generative Workshop

The user generates literary fragments, essays, aphorisms, conceptual drafts, poetic variations, critical readings, reversals, and manifestos from selected atlas paths or regions.

Every generation should include provenance, constraints, central tension, critique, revision handles, and a way to promote useful output back into the canon.

## Key Gaps to Solve

- **Memory is not storage**: The system needs a living interpretive profile containing taste, tensions, motifs, rejected directions, and user judgments.
- **Metadata cannot be generic tags**: It must include rhetoric, poetics, narratology, style, affect, worldview, influence role, and generative affordance.
- **Graph must be creative, not decorative**: Nodes and edges should support generative routes through contradiction, analogy, influence, and border zones.
- **Generation cannot be a terminal action**: It needs drafting, critique, variants, comparison, revision, rejection, and promotion back into canon.
- **Evaluation must affect future behavior**: User taste feedback should influence retrieval, constraints, and generation defaults.

## Advanced Metadata Dimensions

- Bibliographic: title, author, translator, edition, publication year, language, genre, movement, period.
- Passage provenance: source location, page, chapter, section, quote boundary, import source, user annotation, confidence.
- Textual unit type: quote, aphorism, scene, image, argument, question, metaphor, observation, contradiction, maxim.
- Rhetorical form: paradox, antithesis, analogy, reversal, compression, escalation, irony, confession, attack, lament.
- Poetic texture: cadence, density, imagery, abstraction, sonic patterning, sentence shape, figurative load.
- Narrative features: speaker, focalization, temporal structure, scene type, implied audience, distance.
- Worldview coordinates: order/chaos, agency/fate, irony/sincerity, transcendence/materialism, solitude/community.
- Affective register: awe, dread, contempt, tenderness, melancholy, ecstasy, resignation, comic detachment.
- Intertextual relation: echoes, rebuts, extends, descends-from, inverts, parodies, translates, secularizes, radicalizes.
- Influence role: source of image, argument, rhythm, stance, contradiction, title, metaphor, or form.
- Generative affordance: good for aphorism, essay seed, dialogue, prose poem, title, scene premise, critique, counterpoint.
- Risk metadata: quote leakage risk, style mimicry risk, copyright sensitivity, attribution requirement.
- User relationship: loved, resisted, misunderstood, formative, guilty influence, recurring obsession, unresolved tension.

## Literary Quality Criteria

- Worldview fidelity
- Transformative originality
- Rhetorical force
- Image and metaphor precision
- Intertextual integrity
- Stylistic distinctiveness
- Interpretive depth
- Constraint adherence
- Revision usefulness
- Quote leakage safety

## Failure Modes to Avoid

- Generic profundity
- Voice laundering
- Influence mimicry
- Quote leakage
- False lineage
- Tension flattening
- Over-tagging without insight
- Loss of authorial agency
- Treating the user as output curator instead of author

## Guiding Principle

Build a system that helps the user collect, interpret, map, constrain, generate, revise, and reincorporate literary material into a living personal canon.
